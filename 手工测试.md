# 手工测试

1. 阅读并了解软件项目文档/指南。此外，如果可用，请研究被测应用程序（AUT）。
2. 涵盖文档中提到的所有要求的测试用例草稿。
3. 与团队负责人，客户一起审查测试案例并确定基线
4. 在AUT上执行测试用例
5. 报告错误。
6. 修复错误后，再次执行失败的测试用例以验证它们是否通过。
